# 历史粘贴板 PRD（MVP）

## 1. 一句话产品定位

面向不懂代码的 Windows 用户，提供本机历史剪贴板，一键找回最近复制的文字/图片并再次粘贴，避免反复切换窗口翻找。

## 2. 核心用户故事

1. As a Windows 用户，I want 静默记录我复制的文字和图片，so that 忘记内容时能事后找回。
2. As a 忙碌用户，I want 按时间浏览、搜索、置顶/删除历史卡片并可一键复制，so that 快速复用近期内容。
3. As a 注重隐私的用户，I want 数据只存本机且在关机或从托盘退出后清空，so that 不会留下长期或云端痕迹；误关窗口不会丢记录。

## 3. MVP 功能列表

| # | 功能 | 优先级 |
|---|------|--------|
| 1 | 自动监听剪贴板（文字 + 图片）并写入本机历史 | P0 |
| 2 | 历史卡片按时间降序列表；点击条目写回剪贴板 | P0 |
| 3 | 删除 / 置顶 | P0 |
| 4 | 关键词搜索（仅匹配文字条目） | P0 |
| 5 | 浅粉底、列表为主的简洁 UI | P1 |

### 范围约束（工程）

- UI：Next.js 14 + Tailwind + shadcn/ui
- 桌面壳：Electron（本机真剪贴板监听）
- 数据：仅本机会话内存；**关闭窗口 → 托盘后台继续**；托盘「退出」或关机时清空；超过 24 小时的条目自动丢弃
- Demo：约 20 分钟可本地跑通；单文件 ≤200 行

## 4. 验收标准

| 功能 | 可测标准 |
|------|----------|
| 文字监听 | 复制一段文字后 ≤1s 出现在列表顶部，内容与原文一致 |
| 图片监听 | 截图/复制图片后出现带缩略图的卡片；点击后再粘贴为图片 |
| 排序与置顶 | 非置顶区按复制时间严格降序；置顶区固定在上方，组内仍按时间降序 |
| 再次复制 | 点击卡片后，内容写回系统剪贴板，用户可手动 Ctrl+V 粘贴 |
| 删除 | 删除后该条立即消失；同一运行会话内不可再见 |
| 清空 | 关窗进托盘后记录仍在；托盘「退出」或关机后再开，历史为空 |
| 搜索 | 输入关键词只显示文字内容包含该词的条目；图片条目不参与文字匹配 |
| UI | 主色为浅粉系；首屏为单一历史列表，无仪表盘式多模块堆砌 |

## 5. 明确不做的事

- 云同步、登录、Supabase、多设备
- 以 Vercel 或其他公网部署作为运行方式
- 富文本格式保留；文件/文件夹路径复制；Office 专有格式
- 跨开机长期归档（关机或托盘「退出」即清空；关窗不等于退出）
- 复杂设置页、多主题、账号体系
- macOS / Linux / 移动端
- 「一键粘贴到前台窗口」（MVP 只写回剪贴板，由用户自行 Ctrl+V）
- 磁盘持久化历史（当前仅进程内存，后台运行期间保留）

## 6. 关键风险 + 应对

| 风险 | 应对 |
|------|------|
| 纯 Web 无法直读本机剪贴板 | Electron 主进程轮询监听，渲染进程只负责展示；不部署 Vercel |
| 20 分钟 / 单文件 ≤200 行 | 最小骨架：主进程轮询 + 一个历史页；图片用 dataURL；条目上限约 100 |
| 「再次复制」触发自我监听死循环 | 写入剪贴板时短冷却，或用内容指纹去重 |
| 大图占满内存 | 限制单图尺寸或存压缩缩略图；失败则跳过并以文字占位「图片过大」 |

# 开发日日志规范

## 目的

按日记录「已完成 / 待办 / 阻塞」，保证项目小步推进、可回溯，避免一次塞满多阶段工作。

## 目录

- 文件夹：`开发日/`
- 模板：`开发日/_template.md`
- 当日文件：`开发日/YYYY-MM-DD.md`（按本地日期）

## 「自动记录」规则（Agent 纪律）

不做独立定时脚本。由 Cursor Agent 在会话中执行：

### 会话开始

1. 确认今日 `开发日/YYYY-MM-DD.md` 是否存在；不存在则从 `_template.md` 复制并填日期。
2. 读取 `cursor.md`「当前阶段」，与 [04-execution-roadmap.md](./04-execution-roadmap.md) 对齐。
3. 本日待办只服务当前 `D*`，禁止跨阶段堆砌。

### 会话中 / 结束

1. 每完成一个可验收小步，立即勾到「已完成」。
2. 更新「待办」为下一步唯一主线（或本阶段内 1～3 条细项）。
3. 有阻塞写入「风险与阻塞」；需要人决定时明确写出问题。
4. 「明日建议」只写下一小步，不写完整剩余路线图。

## 字段说明

| 字段 | 含义 |
|------|------|
| 日期 | `YYYY-MM-DD` |
| 阶段 | 如 `D0 · 文档体系` |
| 已完成 | 当日实际做完的事项（可测、可复述） |
| 待办 | 下一步未完成项 |
| 风险与阻塞 | 技术/范围/依赖问题 |
| 明日建议 | 仅一条主路径建议 |

## 示例（片段）

```markdown
## 已完成
- [x] 创建 docs/01-requirements.md

## 待办
- [ ] 开始 D1：Electron + Next 最小窗口

## 明日建议
启动 D1 脚手架，验收「能打开应用壳」即停。
```
